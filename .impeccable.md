# openma-desktop

A local-first desktop client for any ACP (Agent Client Protocol) agent —
Claude Agent, Codex CLI, Gemini, OpenCode, Hermes, OpenClaw. Like Claude
Code Desktop but not locked to one model or one vendor.

## Design Context

### Users

Two cohorts in roughly equal weight:

1. **Developers who already use multiple ACP CLIs** (claude / codex / gemini)
   and want one local front-end that doesn't lock them to a vendor or push
   their conversations through a remote SaaS.
2. **Power users / GUI-hesitant types** who'll install a CLI tool if it does
   the job. The UI can't be nerd-only — needs to be friendly enough that
   they don't bounce on first launch.

Important: **the primary surface is general chat**, NOT coding. Most
sessions will be brainstorm / research / writing / quick Q&A. Coding-in-a-
git-repo is one important use case, not the only one. Don't optimize the
shell for IDE-shape (no breadcrumb-heavy file pane, no big right-rail).
Keep the surface anchored on the conversation.

### Brand Personality

Three words: **quiet, dense, intentional**.

- Quiet — refuses the saas-promotional palette. Coral / green / red dot
  ladders read as dashboard chrome. Color appears sparingly and always
  for meaning, never decoration.
- Dense — small type, tight rhythm, lots of whitespace between sections
  but very little within them. Doesn't burn pixels per word.
- Intentional — every element earns its place. Empty space and silence
  are valid choices. Hover/focus reveals interactivity; "look here!"
  glow effects are out.

Adjacent references: **Linear** (clean, neutral, high-density product UI).
Anti-references: dashboard-y SaaS templates with brand-colored status
dots, Web2.0 gradient hero buttons, glassmorphism, generic icon-above-
heading cards.

### Aesthetic Direction

**Light-first AND dark-first** — the user is on macOS / Windows / Linux
all day; theme follows `prefers-color-scheme` by default and only the
system contrast standard matters. Both themes get equal love.

**Brand: muted clay** — `oklch(0.62 0.08 30)` (≈ warm desaturated terracotta).
Carries an heirloom feel from openma console's coral lineage but stripped
of the marketing-red chroma. Never dominates a view; surfaces in row
hover, the agent "breathing" halo while a session runs, the topbar status
pill, the send-button accent. brand-fg = `#2a1a14` for ≥ AA on top of it.

**Type** — Geist Variable (sans body) + JetBrains Mono Variable (mono).
Two families, both variable, no display font. Code blocks + identifiers
in mono; everything else sans. Mono is also the spirit of the `[openma]`
bracket logo — bracket characters render in mono on first paint to avoid
FOUT.

**No status-dot ladder** — agent state lives in icon behavior: idle =
neutral grey, running = muted-clay breathing halo around the agent's
brand mark, errored = the whole row text shifts to `text-danger`. Single
signal per row.

**No hairlines between every section** — surfaces separate via background
tone (the elevation ladder: `bg-sidebar` < `bg-bg` < `bg-bg-surface`),
not by drawing a 1px line between them.

### Design Principles

Five rules that drive every design call here forward:

1. **The icon is the only thing that moves.** A running session breathes
   via its agent icon's halo. No spinner, no pulsing dot, no skeleton
   shimmer in any other surface. The one-thing-moves rule means the eye
   lands on the active conversation, not a Christmas tree.
2. **Color earns its place.** Brand color only on (a) the running-state
   halo, (b) row selection background tint, (c) the send button, (d) the
   focus ring. Errors get danger color, nothing else does.
3. **Border-by-tone, not by hairline.** Elevation ladder carries the
   sectioning. We use `border` only where two surfaces of the same tone
   meet (rare).
4. **First paint feels like landing, not loading.** Every route transition
   uses the `reveal-in` micro-fade (250ms, soft ease). No skeletons, no
   spinners — by the time the user sees the page, it's ready.
5. **Density without claustrophobia.** Sidebar rows at `text-xs` (12px),
   chat body at `text-sm` (14px), max-w-3xl for prose. Vertical rhythm
   uses the 4pt scale; horizontal padding uses the single source
   `--page-pl/pr` token.
